# 燈塔財經 — 設定說明

## 一、n8n 憑證設定

### 1. GitHub PAT（Header Auth）
- Credentials → Add Credential → 選 **Header Auth**
- Name: `authorization`
- Value: `token 你的GitHub_PAT`（注意前面要加 `token ` 空格）
- 儲存名稱：`GitHub PAT - Simple`

### 2. Gemini API Key（HTTP Query Auth）
- Credentials → Add Credential → 選 **HTTP Query Auth**
- Name: `key`
- Value: 你的 Google AI Studio API Key
- 儲存名稱：`Gemini API Key`

### 3. Gmail OAuth2
- Credentials → Add Credential → 選 **Gmail OAuth2**
- 點 Connect my account → 用 ddwang1211@gmail.com 授權
- 儲存名稱：`Gmail - News Notification`

---

## 二、GitHub Repo 建立步驟

1. 到 https://github.com/new
2. Repository name: `lighthouse-daily`
3. 設為 **Public**
4. 勾選 "Add a README file"
5. 點 Create repository

建立後：
6. 進入 repo → Settings → Pages
7. Source 選 **Deploy from a branch**
8. Branch 選 **main**，資料夾選 **/ (root)**
9. 儲存

你的網站網址將會是：
`https://huanyu2001.github.io/lighthouse-daily/`

---

## 三、初始化 Repo（上傳初始檔案）

在建立 repo 後，需要先上傳初始的靜態檔案。
請把以下檔案上傳到 repo 根目錄：
- index.html
- news-render.js
- archive/index.html
- categories/tech.html
- categories/finance.html
- categories/stock.html

上傳方式：在 GitHub repo 頁面點 "Add file" → "Upload files"

之後 n8n 會自動更新：
- news-data.json
- archive-list.json
- index.html
- archive/index.html
- categories/tech.html
- categories/finance.html
- categories/stock.html

---

## 四、匯入 n8n 工作流程

1. 進入 n8n → 左上角選單 → Import
2. 選擇 `n8n-workflow.json`
3. 確認節點連線正確
4. 將三組 Credentials 綁定到對應節點
5. 先用 **Test Workflow** 手動執行一次確認
6. 執行後先檢查 `news-data.json` 是否有建立，然後再打開首頁與各分類頁確認內容有載入

---

## 五、時區說明（重要）

n8n 的排程觸發器使用 UTC 時間：
| 台灣時間 | UTC 時間 |
|---------|---------|
| 07:00   | 前日 23:00 |
| 12:00   | 04:00   |
| 18:00   | 10:00   |

Cron 表達式：
- 早報：`0 23 * * *`
- 午報：`0 4 * * *`
- 晚報：`0 10 * * *`
