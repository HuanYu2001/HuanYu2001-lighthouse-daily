# 燈塔財經 n8n 設定說明

## 1. 需要的 Credentials

### GitHub PAT - Simple
- 類型：`Header Auth`
- Name：`Authorization`
- Value：`token 你的_GitHub_PAT`
- Credential 名稱請設成：`GitHub PAT - Simple`

GitHub PAT 至少需要：
- `repo`

### Gemini API Key
- 類型：`HTTP Query Auth`
- Name：`key`
- Value：`你的 Google AI Studio API Key`
- Credential 名稱請設成：`Gemini API Key`

## 2. n8n Workflow 說明

目前這份 workflow 會做三件事：
- 用 Gemini + Google Search 找出當期新聞
- 更新 `news-data.json`
- 更新 `archive-list.json` 與新增一頁封存 HTML

它**不再改寫**首頁與分類頁 HTML，因為現在網站前端會直接讀 `news-data.json`，這樣更穩定、也比較不容易壞掉。

## 3. 匯入後正常看到的狀態

匯入 workflow 後：
- `Code` 節點不應該出現紅色錯誤叉叉
- `HTTP Request` 節點如果還沒綁定 credentials，會有警示圖示，這是正常的

如果你看到紅色警示，通常代表：
- 還沒設定 `GitHub PAT - Simple`
- 還沒設定 `Gemini API Key`

## 4. 匯入與測試步驟

1. 在 n8n 使用 `Import from file`
2. 選擇 `n8n-workflow.json`
3. 依序綁定：
   - `GitHub PAT - Simple`
   - `Gemini API Key`
4. 先手動執行一次 `Execute workflow`
5. 執行成功後，到 GitHub repo 檢查：
   - `news-data.json`
   - `archive-list.json`
   - `archive/日期-版次.html`
6. 最後回網站重新整理確認內容有顯示

## 6. 排程時間

這份 workflow 使用 UTC cron，對應台灣時間如下：

| 台灣時間 | UTC cron |
|---------|----------|
| 07:00   | `0 23 * * *` |
| 12:00   | `0 4 * * *` |
| 18:00   | `0 10 * * *` |

## 7. 如果執行後網站沒更新

請優先檢查：
- GitHub PAT 是否有 `repo` 權限
- Gemini API Key 是否有效
- workflow 是否有成功寫入 `news-data.json`
- GitHub Pages 是否已從 `main` branch 的 `/ (root)` 發佈
